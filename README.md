# Subscription Aggregation Service

REST-сервис для агрегации данных об онлайн-подписках пользователей. Спроектирован с использованием Clean Architecture (Чистой Архитектуры), ориентирован на высокую производительность и соответствует индустриальным стандартам разработки Middle+.

## Технологический стек
- **Language**: Go 1.26+ (Structured logging через встроенный `slog`, strict линтинг)
- **Database**: PostgreSQL 18 (Работа через пул соединений `pgxpool`, без ORM оверхеда)
- **Routing**: `go-chi/chi/v5` (Безопасное извлечение IP с защитой от спуфинга, автоматический recoverer от паник)
- **Migrations**: `golang-migrate` (Автоматический накат схем данных при старте docker-compose)
- **Documentation**: Swagger / OpenAPI (Интегрированный прямо в приложение Swagger UI)
- **CI/CD**: GitHub Actions (Автоматический strict статический анализ через `golangci-lint` и тесты на состояния гонок)

## Быстрый запуск (Docker Compose)

Сервис полностью контейнеризирован. Инициализация базы данных, проверка доступности СУБД и автоматическое применение миграций происходят прозрачно при старте.

1. Склонируйте репозиторий и перейдите в корень:
   ```bash
   git clone https://github.com/kgdevgo/subscription-api.git
   cd subscription-api
   ```
2. Запустите всё окружение одной командой:
    ```bash
    docker-compose up --build
    ```
   После успешного поднятия контейнеров API сервиса будет доступно локально на порту :8080.

## Спецификация API & Swagger UI

Интерактивная документация Swagger UI вмонтирована прямо в рантайм приложения. Вы можете протестировать эндпоинты через браузер по адресу:
http://localhost:8080/swagger/index.html

### Основные ручки (REST API):
- POST /api/v1/subscriptions — Создание записи о подписке

- GET /api/v1/subscriptions/{id} — Получение информации о подписке по ID

- PUT /api/v1/subscriptions/{id} — Полное обновление записи о подписке

- DELETE /api/v1/subscriptions/{id} — Удаление записи о подписке из системы

- GET /api/v1/subscriptions/total — Агрегация стоимости всех подписок за выбранный период (выполняется эффективно на стороне СУБД)

Пример запроса на создание подписки (POST):
```JSON
{
  "service_name": "Yandex Plus",
  "price": 400,
  "user_id": "60601fee-2bf1-4721-ae6f-7636e79a0cba",
  "start_date": "07-2025"
}
```

Пример запроса на расчет стоимости за период (GET):
```
GET /api/v1/subscriptions/total?user_id=60601fee-2bf1-4721-ae6f-7636e79a0cba&service_name=Yandex%20Plus&from=01-2025&to=12-2025
```

## Конфигурация (.env)
Все конфигурационные параметры вынесены из кода приложения. Локальные переменные и демонстрационный DSN описаны в файле .env.example. При работе в Docker-окружении секреты и доступы безопасно инжектятся из локального .env файла, добавленного в .gitignore.
